流水线时间计算

m端流水线，n个指令，时钟周期为t，总时间为 $m+(n-1)t$

经常于低位交叉存储结合起来考。2012真题，P310 .6

单/双缓冲区时间

原则：

  1. C和M不能同时发生，且C结束紧跟着M：CPU处理完才能接收
  2. 单缓冲下T和M不能同时发生：因为T和M都用到了同一个缓冲
  3. 由1、2可知，单缓冲下只能T和C同时发生

画图分析时假设一个初始状态，计算下次达到这种状态的时间

对于单缓冲来说，初始是缓冲区满

对于双缓冲来说，初始是一个空一个满

注意：若题中要计算从读入到分析数据的总时间，要从缓冲区为空开始算起，计算公式为：第一个数据时间+（n-1）x上面计算的花费时间

大端小端

人看的是小端，机器存的是小端。（小人）

小端：12 34 56，12存储在低位。大端相反。

分页确定逻辑地址结构

  • 虚拟存储器最大容量：看MAR有多少位

  • 页号+页内偏移。页号包括页表索引

  • 多级页表逻辑地址确定：

    页内偏移由页大小决定

    最高一级页表位数不能超过log(一页的页表项数)，其他级的位数都为log(一页的页表项数)

    eg：48位虚地址，页面大小为4KB，页表项大小为8B

    页面大小为4KB，所以页面偏移为12，一页的页表项数 = $4KB/8B = 2^{9}$，剩下的48-12=36位按每级9位分配，36/9=4，所以4级

  • cache组相连映射确定地址结构：

    标记+组号+偏移

    偏移由页大小确定，组号由组数确定，组数=cache条数/m路，剩下的是标记

TLB、页表、cache

cpu根据虚拟地址中的虚拟页号去TLB中找页，若存在将虚拟地址转换为物理地址。根据物理地址去cache中找块，若找到无需到主存中找。

  • 页表是为了实现虚拟内存，完成对内存的扩容。cache是为了解决cpu与主存速度不匹配的问题
  • 缺页要从磁盘中取出页到内存，cache不命中要从主存去取出块到cache。因此缺页开销更大
  • TLB表项为虚拟页号+物理页号（主存）。cache表项为标记+数据。快表是慢表的一个副本，结构一样。cache中存放了数据，因此取数时最少访问主存0次。

删除：

1. 直接删除
 2. 兄弟够借：若被删除节点右（左）兄弟够借，先将共同父亲放入被删除节点，再把右（左）兄弟的最左（右）边的孩子放入父节点。

邻接矩阵、邻接表、邻接多重表、十字链表定义和画法

邻接矩阵

顶点表是一维数组，边表是二维数组

邻接表

分为顶点节点和边表节点，每个顶点节点是一个单链表，连接着与这个顶点相连的边节点（有向图则与出边相连）

十字链表

十字链表是有向图的链式存储

顶点节点：

  • data：顶点符号/编号
  • firstin：第一个入边
  • firstout：第一个出边

弧节点：

  • tailvex：弧尾节点，若一条弧从1指向4，则弧尾节点为1
  • headvex：弧头节点，若一条弧从1指向4，则弧尾节点为4
  • hlink：下一条弧头节点相同的弧
  • tlink：下一条弧尾节点相同的弧


十字链表画法：先画同一行（弧尾相同)，再垂直连接弧头相同的边，最后连接顶点的第一个入边弧并补全空符号。

邻接多重表

是无向图的链式表示，无向图的邻接矩阵中，一条边要占2个格子，删除的时候需要分别修改两个位，邻接多重表中有几条边就有几个弧节点。

顶点节点：

  • data：顶点符号
  • firstedge：第一条边（一般是序号最小）

弧节点：

  • ivex，若一条弧连接0-1，ivex为0
  • ilink，与ivex相连的下一条弧
  • jvex，若一条弧连接0-1，jvex为1
  • jlink，与jvex相连的下一条弧

  • DFS、BFS

最小生成树MST：Prim算法、Kruskal算法

Prim（选点，Point，PP）：任选初始点，选择与此点相连边中权值最小的边，不断重复，直到将所有点加入。复杂度$O(V^2)$

Kruskal（选边）：不断选择权值最小的边，直到连通分量为1。因为用最小堆实现每次选择最小的边，所以复杂度$O(Elog_2(E))$

拓扑排序

结合工程背景，用节点表示先后顺序。选择某种数据结构，基于该数据结构来解决问题。eg：工厂流水线

拓扑序列：陆续删除入度为0的节点。

有可能考算法：indegree数组 or DFS

indegree实现法：

  1. 用邻接表实现简单
  2. 遍历所有边，统计每个顶点的入度并存储在 indgree 中
  3. 将所有入度为0的点入栈
  4. 栈顶元素出栈，遍历其所有出边（这里用邻接矩阵方便），将出边指向的顶点入度-1，入度为0则进栈
  5. 栈为空停止，此时如果还有顶点未输出则有回路

关键路径：AOE网、最早发生、最晚发生、最早开始、最迟开始

若只需要知道关键路径可以看：求解关键路径

最早发生ve：从前往后，选择大的。对于V4: 3+2 < 2+4，所以ve(4) =6

最晚发生vl：从后往前，选择小的。对于V2: 7-3>6-2，所以vl(2)=4

最早开始：对于图中的边a3，从V2指向V4，因此e(3) = ve(2) = 3

最晚开始：对于图中的边a3，从V2指向V4，因此l(3) = vl(4) - weight = 6-2=4

计算机网络

极限传输速率

无噪声，奈氏定理$V = f \times log_{2}N$，f为带宽

有噪声，香农$V = Wlog_2(1+\frac{S}{N})$ W为带宽，S/N为信噪比

带宽单位为MHz，为最高频率-最低频率

eg：一个传输数字信号的模拟信道的功率是0.62W，噪声功率是0.02W，频率范围是3.5～3.9MHz，该信道最高数据传输率是：

有噪声，用香农公式，带宽为0.4MHz，$V = 0.4 \times log_2(1+31) = 2Mbps$

有效数据率/信道利用率

首先要知道发送周期指的是：T=发送延时+RTT，若在T时间内发送mbit数据，则有效数据率$V=\frac{m}{T}$​

设：

  • 帧大小：L

  • 两主机间单向传播时间：t，则RTT为2t

  • 信道带宽/数据传输率：M

  • 发送周期：$T = \frac{L}{M} + 2t + (确认帧发送时间)$​​

    如果题目忽略确认帧发送时间就不算括号内的，如果题目中说确认帧由数据帧捎带，意思是确认帧的发送时间和数据帧发送时间一样为$\frac{L}{M}$​, 王道P81

随机访问

CSMA

CSMA/CD

截断二进制指数退让算法：

  1. 确定基本退让时间$2\tau$
  2. $k = min[重传次数, 10]$
  3. 从$[0, 1, …, 2^k-1]$中选一个数$r$，退避时间为$r.2\tau$
  4. 重传16次抛弃此帧

最小帧长：$总线传播时延 \times 数据传输率 \times 2 = 2\tau v$

  • 以太网规定最短帧长为64B，凡长度小于64B的都是由于冲突而异常终止的无效帧
