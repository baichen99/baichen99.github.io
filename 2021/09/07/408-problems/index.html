多道程序、多任务

多道程序设计是并行处理的一种基本形式，意味着多个进程同时在一个处理器上运行。在现代操作系统中，当多个程序或进程在单个处理器上运行时，操作系统的工作就是有效地管理所有进程。

多道程序特点：

  • CPU利用率高
  • 系统吞吐量达
  • IO设备利用率高：多个程序使用IO设备，IO设备一直工作
  • 并发、共享（单道程序是封闭）、虚拟、异步

多任务是指操作系统使用多个CPU一次执行多个任务的能力。它基本上在一个系统中使用两个或多个CPU来分配共享公共资源（包括CPU和内存）的任务。

多任务特点：

  • 并行：CPU和I/O设备并行
  • 并发：同一时间间隔内执行多个任务
  • 对共享资源进行保护
  • 单个CPU也可满足（2018 23）

多长时间响应一次才能不丢失数据

对于定时查询，查询时间间隔必须小于传输（准备）数据时间

对于中断查询，中断响应时间小于传输（准备）数据时间

1）设备A每次传输数据时间=$\frac{32bit}{2MB/s} = 2us$，必须每2us查询一次，

因为传输速率为2MB/S，所以每秒查询$\frac{2MB}{4B}=0.5M$次，每次10条指令，IO时间占CPU总时间=$\frac{0.5M \times 10 \times 4}{500M}= 4\%$

2） 设备B传输数据时间=$\frac{32bit}{40MB/s} = 0.1us$ 中断处理时间=$400 / 500MHz = 0.8us$ 因为中断处理时间大于数据传输时间，数据会被刷新，造成丢失

3）DMA方式使用CPU的时间只有预处理和后处理，每次传1000B，一秒传$\frac{40MB}{1000B}=40000$次，百分比=$\frac{40000 \times 500}{500M}=4\%$

IO时间占CPU占用百分比计算方法必须掌握，真题2019 21

  1. 关中断
  2. 保存断点

  • 中断传输数据以字节为单位，

DMA过程

DMA分为预处理、数据传送、后处理3个阶段

预处理：由CPU完成必要的准备工作，数据传送前由DMA控制器请求总线使用权

数据传送：由DMA控制器直接控制总线完成

后处理：DMA控制器向CPU发送中断请求

通道过程

  1. CPU通过管理程序组织通道程序，并使用I/O指令启动通道
  2. 通道处理器执行通道程序，完成IO操作
  3. 通道程序结束后，向CPU发出中断请求

一次突发传送总线事务时间=送首地址和命令时间（一个总线时钟周期）+存储器准备数据时间+传送数据时间

存储器准备数据时间即为8体交叉存储器传送32B数据时间，因为一个体字长32b，8个体把存储周期分为8份，每份40/8=5ns，8个体需要传$[8+(8-1)]*5 = 75ns$

每准备好32b就把它传送出去，传送时间为1个时钟周期 =5ns，第二个准备好时，第一个刚好传送完，以此类推，最后第8个传送完后再花5ns

所以总时间= 5+75+5

内核级线程、用户级线程

用户级线程

  1. 线程的调度不需要内核直接参与，控制简单。
  2. 可以在不支持线程的操作系统中实现。

内核级线程

  1. 线程在用户态的运行，而线程的调度和管理在内核实现，在控制权从一个线程传送到另一个线程需要用户态到内核态再到用户态的模式切换，比较占用系统资源。

区别和联系

  1. 内核支持线程是操作系统内核可感知的，而用户级线程是OS内核不可感知的。
  2. 用户级线程的创建、撤消和调度不需要OS内核的支持。
  3. 用户级线程只能运行在用户态下，而内核支持线程的程序则是可以运行在任何状态下的程序。

open、read、close系统调用

  1. open根据（路径、文件名）找到相应的目录文件，并把目录文件读入内存，从目录中找到文件名对应的目录项，并检查权限。将目录项复制到内存中的打开文件表，返回文件描述符
  2. read根据文件描述符找到对应FCB，将文件内容读入内存，返回数据缓冲区的指针给用户进程
  3. close把文件当前的控制信息从内存写回外存

分页确定逻辑地址结构

  • 虚拟存储器最大容量：看MAR有多少位

  • 页号+页内偏移。页号包括页表索引

  • 多级页表逻辑地址确定：

    页内偏移由页大小决定

    最高一级页表位数不能超过log(一页的页表项数)，其他级的位数都为log(一页的页表项数)

    eg：48位虚地址，页面大小为4KB，页表项大小为8B

    页面大小为4KB，所以页面偏移为12，一页的页表项数 = $4KB/8B = 2^{9}$，剩下的48-12=36位按每级9位分配，36/9=4，所以4级

  • cache组相连映射确定地址结构：

    标记+组号+偏移

    偏移由页大小确定，组号由组数确定，组数=cache条数/m路，剩下的是标记

TLB、页表、cache

cpu根据虚拟地址中的虚拟页号去TLB中找页，若存在将虚拟地址转换为物理地址。根据物理地址去cache中找块，若找到无需到主存中找。

  • 页表是为了实现虚拟内存，完成对内存的扩容。cache是为了解决cpu与主存速度不匹配的问题
  • 缺页要从磁盘中取出页到内存，cache不命中要从主存去取出块到cache。因此缺页开销更大
  • TLB表项为虚拟页号+物理页号（主存）。cache表项为标记+数据。快表是慢表的一个副本，结构一样。cache中存放了数据，因此取数时最少访问主存0次。

邻接矩阵

顶点表是一维数组，边表是二维数组

邻接表

分为顶点节点和边表节点，每个顶点节点是一个单链表，连接着与这个顶点相连的边节点（有向图则与出边相连）

十字链表

十字链表是有向图的链式存储

顶点节点：

  • data：顶点符号/编号
  • firstin：第一个入边
  • firstout：第一个出边

弧节点：

  • tailvex：弧尾节点，若一条弧从1指向4，则弧尾节点为1
  • headvex：弧头节点，若一条弧从1指向4，则弧尾节点为4
  • hlink：下一条弧头节点相同的弧
  • tlink：下一条弧尾节点相同的弧


十字链表画法：先画同一行（弧尾相同)，再垂直连接弧头相同的边，最后连接顶点的第一个入边弧并补全空符号。

邻接多重表

是无向图的链式表示，无向图的邻接矩阵中，一条边要占2个格子，删除的时候需要分别修改两个位，邻接多重表中有几条边就有几个弧节点。

顶点节点：

  • data：顶点符号
  • firstedge：第一条边（一般是序号最小）

弧节点：

  • ivex，若一条弧连接0-1，ivex为0
  • ilink，与ivex相连的下一条弧
  • jvex，若一条弧连接0-1，jvex为1
  • jlink，与jvex相连的下一条弧

最小生成树MST：Prim算法、Kruskal算法

Prim（选点，Point，PP）：任选初始点，选择与此点相连边中权值最小的边，不断重复，直到将所有点加入。复杂度$O(V^2)$

Kruskal（选边）：不断选择权值最小的边，直到连通分量为1。因为用最小堆实现每次选择最小的边，所以复杂度$O(Elog_2(E))$

最小生成树唯一的条件：所有边的权值不同

极限传输速率

无噪声，奈氏定理$V = 2f \times log_{2}N$，f为带宽

有噪声，香农$V = Wlog_2(1+\frac{S}{N})$ W为带宽，S/N为信噪比

带宽单位为MHz，为最高频率-最低频率

若给出信噪比为30dB，需要将其转换为S/N，关系是$30 = 10log_{10}1000$,所以S/N=1000

eg：一个传输数字信号的模拟信道的功率是0.62W，噪声功率是0.02W，频率范围是3.5～3.9MHz，该信道最高数据传输率是：

有噪声，用香农公式，带宽为0.4MHz，$V = 0.4 \times log_2(1+31) = 2Mbps$

GBN传输速率、信道利用率

GBN中$信道利用率 = \frac{有效时间}{周期} = \frac{传输N帧时间}{收到第一个确认帧时间}$

传输速率=$\frac{N\times 帧长}{收到第一个确认帧的时间}$

GBN在接收到一次确认帧之前可以发N个帧。

此题发送一个确认帧时间为50x2=100ms，N个帧大小为1000x1000B=1MB，1MB/100ms=80Mbps

GBN中N越大，信道利用率越高，根据信道利用率公式，N越大，帧长越小，所以选每帧128B

现在要求一次发多少帧，从而算出比特数

一次即收到第一个确认帧之前这段时间：即$\frac{128 \times 8}{16k} \times 2 \times 1000 + 270 \times 2 = 668ms$

这段时间可以发$668ms \times 16kbps / 128B=10.4$ 所以可以发10帧，比特数至少为4

有效数据率, 信道利用

  • 帧大小：L

  • 两主机间单向传播时间：t，则RTT为2t

  • 信道带宽/数据传输率：M

  • 发送周期：$T = \frac{L}{M} + 2t + 确认帧发送时间$​​

如果题目忽略确认帧发送时间就不算括号内的，如果题目中说确认帧由数据帧捎带，意思是确认帧的发送时间和数据帧发送时间一样为$\frac{L}{M}$​, 王道P81

发送周期：从开始发送第一个帧开始到收到第一个确认帧的时间，不是所有的

信道利用率=数据发送时间/发送周期

信道吞吐率=发送速率x信道利用率

例题：若最大帧长1000B，用3bit编号，RTT=0.96ms，信道传输率为100Mb/s，捎带确认，求最大信道利用率

​ 3bit编号，发送窗口最大为8-1=7，所以一次最多发7000B

​ 发送周期=发送时间+传输时间+确认帧发送时间=$\frac{1000B}{100Mb/s} + 0.96 + \frac{1000B}{100Mb/s}$

​ 数据发送时间=$7 \times \frac{1000B}{100Mb/s}$

设数据帧长度=x，数据传输时间=$\frac{x}{3kbps}$，信道利用率=$\frac{x/3k}{200ms \times 2 + x/3k}=0.4$解得x=800bit

CSMA/CD

截断二进制指数退让算法：

  1. 确定基本退让时间$2\tau$
  2. $k = min[重传次数, 10]$
  3. 从$[0, 1, …, 2^k-1]$中选一个数$r$，退避时间为$r.2\tau$
  4. 重传16次抛弃此帧

最小帧长：$总线传播时延 \times 数据传输率 \times 2 = 2\tau v$

注意传播和传输的区别，数据大小/传输速率=发送时延

  2. H1通过ARP获得R1的MAC地址，填入帧中发送

  4. R1通过ARP得到H2的MAC地址，构造帧发送到H2所在网络
  5. H2得到IP数据报

GBN和TCP区分

  • GBN中$ACK_n$表示对n号帧的确认，期望收到n+1号帧
  • TCP中$ack_n$表示对n-1号帧的确认，期望收到n号帧

在GBN中，发送方需要重发出错帧及之后的帧

GBN中一个窗口包含一个帧，TCP一个窗口为一个MSS

要注意区分GBN、SR、TCP协议的确认机制:

  1. GBN和TCP都采用累计确认机制，SR协议不是

  2. 关于ACK序号的问题:

    i. GBN:ACK N表示N及其之前的都正确接收

    ii. SR:ACK N表示正确接收N号帧
    iii. TCP:ACK N表示期待收到N号帧

慢开始、拥塞避免、快恢复、快重传区分

  • 慢开始、快恢复：当遇到超时执行慢开始即窗口设为1，sshthresh/=2
  • 当连续收到三个重复确认执行快恢复，窗口/=2，sshtresh/=2
  • 拥塞避免按线性增长
